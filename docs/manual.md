# 交流会 運営マニュアル

このシステムを使って交流会1回分の運営を進めるための手順書です。
**運営者が変わってもこの通りに進めれば同じ運営ができる**ことを目的にしています。

> **ログインID・パスワードについて**: このページの上部に表示されているカードを参照してください(リポジトリでこのファイルを直接読んでいる場合は、Vercel の環境変数 `ADMIN_USER` / `ADMIN_PASSWORD` が実物です)。

> ⚠️ **このページのURLの扱い**: このマニュアルのURLは「知っている人は誰でも開ける」仕組みです。ログイン情報まで見えるので、**運営メンバー以外に転送しない**でください。もし流出したら、Vercel の環境変数 `DOCS_TOKEN` を変更して再デプロイすればURLが無効になります(あわせて `ADMIN_PASSWORD` の変更も検討)。

## 1. このシステムでできること

- **日程調整の自動化** — 調整さん(chouseisan.com)に来月全日程の出欠表を作ってグループに投稿し、集計上位2日程でイベントを自動作成
- 開催アナウンス(参加ボタン付き)の送信と**参加者の自動集計**
- 日程別LINEグループの案内送信
- 自己紹介スライドの記入依頼
- **前日15:00・当日9:00 の案内の自動送信**
- **終了後アンケートの自動送信**
- 「どこまで対応したか」が一目でわかる**チェックリスト**
- 複数LINEチャネル対応(無料枠のグループ分散)とチャネル別の消費量表示

人がやるのは「ボタンを押す」「LINEグループを作る」「スライドURLを貼る」くらいで、送信作業や集計はシステムが行います。

## 2. 管理画面へのログイン

### 2.1 開き方

1. このページ上部のカードにある **管理画面URL** を開く
2. ブラウザがユーザー名とパスワードを聞いてくるので、カードの **ID / パスワード** をそのまま入力する
3. 一度入力すればしばらくは聞かれません。スマホ・PCどちらでも使えます

💡 管理画面URLはブックマーク(スマホならホーム画面に追加)しておくと便利です。

### 2.2 ログインできないとき

| 症状 | 対処 |
|---|---|
| 入力ダイアログが何度も出る | ID/パスワードの打ち間違いです。このページのカードからコピーして貼り付けてください |
| 「500」などのエラーが出る | Vercel の環境変数 `ADMIN_USER` / `ADMIN_PASSWORD` が消えている可能性。Vercel にログインできる人に確認を依頼 |

## 3. 月次の運営フロー(全体像)

```
⓪日程調整(調整さん・ほぼ自動) → ①イベント作成(取込で自動) → ②アナウンス送信 → (参加ボタンで自動集計)
→ ③日程別グループ作成・紐付け → ④グループ案内送信 → ⑤スライドURL設定・案内送信
→ 前日15:00 自動送信 → 当日9:00 自動送信 → アンケート自動送信 → ⑥完了にする
```

⚙️ 日程調整・イベント作成・前日・当日・アンケートは**ほぼ自動**です。人の作業は②〜⑤のボタン操作が中心で、進捗はすべてイベント詳細ページの**チェックリスト**に表示されます。

### 3.1 ⓪ 日程調整をする(調整さん)

1. 管理画面 → **日程調整** → **+ 来月の日程調整を開始**
   → 調整さん(chouseisan.com)に来月の全日程の出欠表が自動作成され、URLがメイングループにLINE投稿されます
2. メンバーは調整さんページで ◯/△/× を入力します(アプリのインストールやログインは不要)
3. 回答が集まったら **結果を取り込んでイベント作成** を押す
   → ◯=1点・△=0.5点で集計した**上位2日程**(同点は早い日付優先)のイベントが自動作成され、イベント詳細に移動します

💡 開始時刻は19:00で作成されます。変更したい場合は日程カードの「開催日時」を編集してください。
💡 「⚠️ 未投稿」バッジが出ていたら、URLのLINE投稿だけ失敗しています。「グループに投稿する」で再送できます。

### 3.2 ① イベントを作成する(手動で作る場合)

⓪の取込を使わず手動で作ることもできます:

1. 日程調整の結果を見て、参加人数が多い**2日程**を決める(最終判断は運営者)
2. 管理画面 → イベント一覧 → **+ 新しいイベント**
3. イベント名(例: 7月交流会)と2つの日程・開始時刻を入力して作成

→ どちらの方法でも、作成時点で「前日15:00」「当日9:00」「アンケート(デフォルト当日21:00)」の自動送信が予約されます。

### 3.3 ② 開催アナウンスを送る

1. イベント詳細のチェックリストで **開催アナウンス(参加ボタン付き)** の「送信する」を押す
   (メイングループが未設定だと警告が出ます。初回セットアップはREADME参照)
2. メイングループに日程ごとの**参加ボタン付きメッセージ**が届く
3. メンバーがボタンを押すと、トークには何も表示されずに**参加者リストへ自動で記録**されます(取り消しボタンもあります)。誰が押したかはイベント詳細の参加者リストで確認してください

💡 ボタンを押しそびれた人・口頭で参加表明した人は、日程カードの「名前を入力して手動追加…」で追加できます。
💡 アナウンスには「参加状況を確認」ボタンが付いていて、メンバーは参加者一覧ページ(閲覧専用)を開けます。

### 3.4 ③ 日程別LINEグループを作って紐付ける

1. LINEアプリで日程ごとのグループを作成する(例: 「7/18 交流会」)
2. 作ったグループに**ボットを招待**する → 管理画面の「グループ」ページに自動で現れる
   ※ 2チャネル構成(グループ分散)の場合は、日程別グループには**チャネル2のボット**を招待する(メイングループはチャネル1のボット)。どちらのチャネルが担当かはグループページに表示され、送信時は自動で使い分けられます
3. イベント詳細の日程カードで「日程別LINEグループ」からそのグループを選んで**保存**
4. LINEアプリでグループの**招待リンク**をコピー(グループ設定 → メンバー招待 → リンクをコピー)し、日程カードの「グループ招待リンク」に貼って保存

### 3.5 ④ グループ案内を送る

チェックリストの **日程別グループ案内** の「送信する」を押す。
メイングループに「7/18(土)参加の方はこちら → 招待リンク」という案内が流れます。

※ 招待リンクから入らない人がいたら、LINEアプリから直接招待してください(ボットにはできない操作です)。

### 3.6 ⑤ 自己紹介スライドを設定・案内する

1. 日程カードの「自己紹介スライドURL」を入力して保存
2. あわせて「参加方法」(Zoomリンクや会場)と「当日の流れ」も入力しておく
   — **当日案内の自動送信に使われます。「参加方法」が未入力だと当日案内が失敗します**(チェックリストに理由が表示されるので、入力して「再送信」すれば大丈夫です)
3. チェックリストの **自己紹介スライド案内** の「送信する」を押す

※ 誰が未記入かの管理はしません。「まだ書いていない人は書いてね」という全体向け文面です。
※ 節約したい月は、この案内を省略しても前日案内にスライドURLが入ります(「5.4 LINE無料枠」参照)。

### 3.7 ⑥ イベントを完了にする

すべて送信し終わったら「完了にする」を押します(一覧の見た目の整理のためで、押し忘れても実害はありません)。

## 4. 各画面の使い方

### 4.1 イベント(ホーム)

全イベントの一覧です。各イベントにステータスが付きます:

| バッジ | 意味 |
|---|---|
| 準備中 | 作成直後。まだアナウンスを送っていない |
| アナウンス済み | 開催アナウンス送信済み。参加受付中 |
| 完了 | 「完了にする」を押したイベント |

### 4.2 イベント詳細(いちばん使う画面)

#### チェックリストの読み方

上から運営フロー順に並んでいます: 開催アナウンス → 日程別グループ案内 → 自己紹介スライド案内 → 前日案内(前日15:00) → 当日案内(当日9:00) → 終了後アンケート。

**状態列**:

| 表示 | 意味 |
|---|---|
| ◯月◯日 ◯:◯◯ に自動送信 | 予約済み。その時刻に自動で送られる |
| 未送信 | 手動送信タスク。ボタンを押すと送られる |
| 送信中… | いま送信処理中。少し待ってから再読み込み |
| ✅ 送信済み | 送信完了(送信時刻も表示) |
| ❌ 失敗 | 送信失敗。**理由が下に表示される**ので、直して「再送信」 |
| スキップ | 送信しない扱いになったもの |

**操作列**:

| ボタン | 意味 |
|---|---|
| 送信する | 手動タスクを送信 |
| 今すぐ送る | 自動送信を予約時刻より前倒しで送信(例: 交流会が終わった直後にアンケートを送りたいとき) |
| 再送信 | 失敗したものを、設定を直したあとにもう一度送る |
| 再送 | 送信済みのものをもう一度送る(同じ内容がもう一度グループに届くので注意) |

#### 日程カード(日程ごとの参加者・設定)

| 項目 | 説明 |
|---|---|
| 参加者リスト | 参加ボタンで自動記録。「手動」バッジは運営者が追加した人。✕で外せる |
| 名前を入力して手動追加… | ボタンを押しそびれた人を追加 |
| 開催日時 | 変更すると**未送信の前日・当日案内の予約時刻も自動で追従**する |
| 日程別LINEグループ | ボット招待済みのグループから選んで紐付け |
| グループ招待リンク | グループ案内の本文に入るリンク |
| 自己紹介スライドURL | スライド案内・前日案内・当日案内の本文に入る |
| 参加方法 | Zoomリンクや会場。**当日案内に必須** |
| 当日の流れ | 任意。当日案内に載る |
| アンケート送信日時 | デフォルトは当日21:00。送信前なら変更できる |

#### イベントの完了・削除

- **完了にする** — 一覧の整理用
- **削除** — 参加者記録・送信予約もすべて消えます。基本使いません

### 4.3 日程調整

- **+ 来月の日程調整を開始** — 調整さんに出欠表を作成し、URLをメイングループに投稿(グループ人数分のメッセージ数を消費)
- **結果を取り込んでイベント作成** — 回答を集計して上位2日程のイベントを自動作成
- 「調整さんで開く」でいつでも回答状況を確認できます
- 取込後にイベントを削除してしまった場合は「結果を取り込み直す」で再作成できます

### 4.4 グループ

ボットをLINEグループに招待すると自動で一覧に載ります。役割を設定してください:

| 役割 | 意味 |
|---|---|
| メイン | 全体アナウンスの送信先。**必ず1つ**設定する |
| 日程別 | 日程に紐付けるグループ |
| 未分類 | 招待直後の初期状態 |

- 「チャネル」列はどのボット(チャネル1/2)がそのグループにいるかで、自動記録されます(手動変更不可)
- 「退出済み」はボットがグループから外された状態です

### 4.5 設定

- **メイングループ** — 設定済みかの確認(設定自体はグループページで行う)
- **LINEチャネルと無料枠** — 当月のメッセージ消費量をチャネル別に表示。月末に足りるかの確認に使う
- **アンケートURL** — 終了後アンケートのリンク先。「回答が1回目の方用」「2回目以降の方用」の2種類

## 5. 自動送信の仕組み

### 5.1 送信タイミング一覧

なにもしなくても以下が送信されます:

| タイミング | 宛先 | 内容 |
|---|---|---|
| 前日15:00 | 日程別グループ | 明日開催・開始時間・スライドURL・未記入者への依頼 |
| 当日9:00 | 日程別グループ | 本日開催・開始時間・参加方法・スライドURL・当日の流れ |
| 当日21:00(変更可) | 日程別グループ | アンケート(1回目用/2回目以降用のURL) |

### 5.2 失敗したときの見え方と再送信

- 送信に失敗するとチェックリストに ❌ と**理由**が表示されます。設定(グループ紐付け・参加方法など)を直して「再送信」を押してください
- 開催日時を変更すると、未送信の予約時刻は自動で追従します
- 「今すぐ送る」で予約より前倒しできます

### 5.3 裏側の仕組み(触る必要はありません)

- 外部サービス cron-job.org が**5分ごと**にこのシステムを呼び出し、期限が来た予約メッセージだけを送信します(誤差は最大5分)
- 同じメッセージが二重送信されない仕組み(冪等)になっています
- 「時刻になっても送られない」ときは cron-job.org のジョブが止まっていないかを確認します(→ 6章)

### 5.4 LINE無料枠とグループ分散(チャネル1/2)

- LINEのメッセージは**グループ宛でもグループ人数分**が月間カウントされます。無料プランは**月200通**
- そのためこのシステムは2つのLINEチャネル(ボット)を使い分けます:
  - **チャネル1** — メイングループ担当(日程調整URL・開催アナウンス・グループ案内)
  - **チャネル2** — 日程別グループ担当(スライド案内・前日・当日・アンケート)
- 36人メイン + 18人×2グループ・月1開催なら各チャネル約144通で無料枠に収まる想定です
- 当月の実測値は**設定ページ**で確認できます。足りなくなりそうな月は、スライド案内の送信を省略すると節約できます(前日案内にもスライドURLが入るため)
- 参加ボタンのタップはメッセージ数を消費しません

## 6. トラブルシューティング

| 症状 | 確認すること |
|---|---|
| 管理画面に入れない | ID/パスワードの打ち間違い(このページ上部のカード参照)。それでもだめなら Vercel の環境変数 `ADMIN_USER` / `ADMIN_PASSWORD` |
| 日程調整の開始・取込が失敗する | 調整さん(chouseisan.com)の仕様変更の可能性。エラーメッセージに理由が出ます。復旧まで調整さんを手動運用し、手動イベント作成(3.2)に切り替える |
| 日程調整のURLがグループに届かない | 日程調整ページに「⚠️ 未投稿」が出ていれば「グループに投稿する」で再送。メイングループ未設定の場合はグループページで設定 |
| ボタンを押したのに参加者に反映されない | Webhook設定(LINE Developersコンソール)が有効か(チャネル2は `?channel=2` 付きURL)。Vercelのログにエラーがないか |
| 前日/当日案内が時刻になっても届かない | cron-job.org のジョブが動いているか(実行履歴を確認)。チェックリストが ❌ になっていれば理由が書いてある |
| チャネル2のグループに送信できない | Vercelの環境変数 `LINE_CHANNEL_2_ACCESS_TOKEN` / `LINE_CHANNEL_2_SECRET` が設定済みか。チェックリストの❌の理由に変数名が出ます |
| グループがグループ一覧に出てこない | ボットがそのグループに入っているか。公式アカウント設定で「グループ・複数人チャットへの参加を許可」が有効か |
| 送信が「メイングループ未設定」で失敗 | グループページでメイングループを「メイン」に設定したか |
| このマニュアルのURLが流出した | Vercel の環境変数 `DOCS_TOKEN` を新しいランダム文字列に変更して再デプロイ。必要なら `ADMIN_PASSWORD` も変更 |

- [ ] このマニュアルのURL(ログイン情報はページ上部に表示される)
- [ ] Vercel・Supabase・cron-job.org・LINE Developers の各アカウント(または招待)
- [ ] メイングループ・ボットの公式アカウントの管理権限
- [ ] 「このマニュアルのURLは運営メンバー以外に転送しない」ルールの共有
